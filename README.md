# 🚗 Buriram Journey Checklist - Web Application (Premium Edition)

เว็บแอปพลิเคชันระบบเช็คลิสความเรียบร้อยของห้องพักและกระเป๋าเดินทางข้ามจังหวัด สำหรับผู้ใช้ที่ต้องการจัดเตรียมสิ่งของอย่างละเอียด คอนฟิกการทำงานทั้งหมดอยู่บนฝั่ง Client-side (Static Site) ที่มีประสิทธิภาพสูง โหลดเร็ว และไม่จำเป็นต้องใช้งานระบบคลาวด์แบ็คเอนด์ที่ยุ่งยาก

* **ใช้งานหน้าเว็บจริง:** [https://bravo0000.github.io/Checklist-Web-App/](https://bravo0000.github.io/Checklist-Web-App/)
* **เทคโนโลยีหลัก:** Vanilla HTML5, Vanilla CSS3 (Custom Properties), Vanilla JavaScript (ES6+), LocalStorage Database, และ Canvas Confetti API

---

## 📂 โครงสร้างโปรเจกต์ (Project Architecture)

โปรเจกต์ประกอบด้วย 3 ไฟล์หลัก ทำงานสอดประสานกันแบบแยกส่วนชัดเจน (Separation of Concerns):

```text
├── index.html     # โครงสร้าง DOM, Semantic Tags, Layout หน้ากาก UI และ Dashboard
├── styles.css     # ดีไซน์ระบบสี HSL, Glassmorphism, Responsive Media Queries, Light/Dark Theme
└── app.js         # ตัวควบคุมสถานะ (State Controller), ตัวจัดการฐานข้อมูล Local Storage และ Event Bindings
```

---

## ⚙️ ระบบจัดเก็บข้อมูลภายใน (Client-side Data Persistence)

แอปพลิเคชันนี้ใช้ **Local Storage** ของเบราว์เซอร์เปรียบเสมือนฐานข้อมูลหลัก เพื่อความรวดเร็วและเป็นส่วนตัวสูงของข้อมูล โดยมี Key ในการจัดเก็บดังนี้:

| Storage Key | รูปแบบข้อมูล (Data Format) | คำอธิบายหน้าที่ |
| :--- | :--- | :--- |
| `buriram_checklist_data_v1` | `Array of Objects` | จัดเก็บสถานะเช็คลิส หมวดหมู่ทั้งหมด รวมถึงรายการที่ผู้ใช้กำหนดขึ้นมาใหม่ (Custom Tasks) |
| `buriram_checklist_theme` | `String` (`"dark"` \| `"light"`) | บันทึกโหมดการมองเห็นเพื่อแสดงผลตามโหมดสีล่าสุดที่ผู้ใช้เลือก |
| `buriram_checklist_notes` | `String` (Plain Text) | บันทึกข้อความสรุปการเดินทาง (Travel Notes) ที่บันทึกอัตโนมัติขณะพิมพ์ |

## 🧠 สถาปัตยกรรมและการคอนฟิกฝั่งโค้ด JavaScript (`app.js`)

สำหรับนักพัฒนาคนอื่น ๆ ที่เข้ามาดูแลหรือพัฒนาโปรเจกต์ต่อ สามารถดูรายละเอียดการคอนฟิกหลักได้ดังนี้:

### 1. โครงสร้างข้อมูลเช็คลิสเริ่มต้น (`DEFAULT_CATEGORIES`)
รายการเช็คลิสเริ่มต้นจะเก็บอยู่ในอาร์เรย์ `DEFAULT_CATEGORIES` แต่ละออบเจกต์ในอาร์เรย์มีโครงสร้าง Schema ดังนี้:

```javascript
{
    id: "power",                // ID ประจำหมวดหมู่ (ต้องไม่ซ้ำกัน)
    title: "ระบบไฟฟ้าและพลังงาน",   // ชื่อแสดงบนการ์ด UI
    icon: "🔌",                 // ไอคอนอิโมจิแสดงกำกับหน้าหัวข้อ
    color: "var(--color-power)", // สีตัวอักษรและไฮไลต์ที่อ้างอิงจาก CSS Variables
    bgColor: "var(--color-power-bg)", // สีพื้นหลังของการ์ด
    tab: "outgoing",            // กำหนดว่าอยู่แท็บใด ("outgoing" = ขาไป / "incoming" = ขากลับ)
    items: [                     // รายการย่อยภายใต้หมวดหมู่นั้น
        { 
            id: "power-1",       // ID ประจำรายการ
            text: "ถอดปลั๊กเครื่องใช้ไฟฟ้า...", // ข้อความแสดง
            checked: false,      // สถานะการขีดเลือก (true = ทำแล้ว / false = ยังไม่ได้ทำ)
            custom: false        // บ่งบอกว่าเป็นรายการเริ่มต้นหรือเพิ่มขึ้นมาเอง (Custom Item)
        }
    ]
}
```

### 2. ระบบรันไทม์หลักและฟังก์ชันสำคัญ

* **`loadData()`**: ทำหน้าที่อ่านข้อมูลจาก `localStorage` หากไม่มีจะทำข้อมูลเริ่มต้นโดยการคัดลอกจาก `DEFAULT_CATEGORIES` ในกรณีที่มีข้อมูลอยู่แล้ว ฟังก์ชันนี้จะประมวลผลการผสานข้อมูล (Safe Synchronization) เพื่อนำข้อมูลเช็คลิสหมวดหมู่หรือรายการใหม่ ๆ ที่พึ่งถูกกำหนดในเวอร์ชันใหม่ไปใส่ลงในฐานข้อมูลเก่าของผู้ใช้ทันที โดยไม่รบกวนข้อมูลเช็คลิสที่เคยติ๊กไว้แล้ว
* **`renderApp()`**: รันการล้างและแสดงผล DOM ของการ์ดเช็คลิสทั้งหมดใหม่ตามแท็บปุ่มปัจจุบัน (`activeTab` ซึ่งมีค่าสลับไปมาระหว่าง `outgoing` และ `incoming`)
* **`updateProgress()`**: คำนวณความคืบหน้าแบบเรียลไทม์ (เป็นเปอร์เซ็นต์ %) **เฉพาะแท็บที่ทำงานอยู่** เพื่อให้แถบ Progress Bar และตัวเลขบอกความก้าวหน้าทำงานแยกกันได้อย่างเด็ดขาด
* **`initNotes()` และการลดการทำงานซ้ำซ้อน (Debounce Saving)**:
  กล่อง Travel Notes เชื่อมต่อกับ Event `input` โดยใช้ตัวตั้งเวลาดีเลย์ (`saveTimeout` ขนาด 800ms) ในการเซฟ เพื่อช่วยลดทราฟฟิกการบันทึกข้อมูลลงฐานข้อมูลโดยตรงอย่างพร่ำเพรื่อ ขณะที่พิมพ์จะเปลี่ยนสถานะ Badge เป็น `"กำลังพิมพ์... ✍️"` และเปลี่ยนเป็น `"บันทึกอัตโนมัติแล้ว 💾"` เมื่อพิมพ์เสร็จ 
* **`fireConfetti()`**: เรียกใช้ฟังก์ชัน `confetti()` จากไลบรารีภายนอก เมื่อค่าความสำเร็จขึ้นครบ 100% ในแต่ละแท็บ เพื่อสร้างเอฟเฟกต์พลุกระดาษฉลองความสำเร็จ โดยมีการบันทึกสถานะ `celebrationTriggered = { outgoing: false, incoming: false }` เพื่อไม่ให้พลุแสดงผลรัว ๆ ซ้ำหากผู้ใช้ติ๊กสลับช่องเล่น

## 🎨 โครงสร้างระบบสไตล์ UI (`styles.css`)

ตัวโปรเจกต์ขับเคลื่อนด้วยระบบดีไซน์ทันสมัยแบบ **Glassmorphism** และมีความยืดหยุ่นสูง:

### 1. ตัวแปรโทนสีหลัก (Color Custom Properties)
สไตล์ชี้เป้าสีต่าง ๆ ถูกผูกไว้กับ CSS Variables ภายใน `:root` ช่วยให้แก้ไขหรือปรับแต่งธีม (Theming) ได้อย่างสะดวก:

```css
:root {
    /* Base Theme System (Default: Dark Mode) */
    --bg-base: #0f121d;
    --card-bg: rgba(22, 28, 45, 0.65);
    --card-border: rgba(255, 255, 255, 0.08);
    --text-primary: #f3f4f6;
    
    /* Category Accent Colors (HSL Tailored Colors) */
    --color-power: HSL(45, 100%, 60%);
    --color-power-bg: rgba(245, 158, 11, 0.1);
    --color-water: HSL(200, 100%, 60%);
    --color-water-bg: rgba(14, 165, 233, 0.1);
    ...
}
```

### 2. ระบบตอบสนองตามอุปกรณ์ (Responsive Breakpoints)
* **Mobile (กว้างไม่เกิน 768px):** แดชบอร์ดจะปรับคอลัมน์ให้อยู่ในแถวเดียวกันอย่างสวยงาม แผ่นการ์ดเช็คลิสต่าง ๆ จะเปลี่ยนการแสดงผลเป็น Grid แบบ 1 คอลัมน์
* **Desktop (กว้างตั้งแต่ 769px ขึ้นไป):** จัดวางรูปแบบ 2 คอลัมน์คู่ขนาน เพื่อใช้งานพื้นที่หน้าจอขนาดใหญ่ให้คุ้มค่าที่สุด

## 🚀 แนวทางพัฒนาต่อและปรับปรุงฟีเจอร์ (Future Enhancements)

โปรเจกต์นี้ได้รับการออกแบบและจัดเตรียมโครงสร้างให้พร้อมพัฒนาต่อยอดร่วมกับเครื่องมือและสถาปัตยกรรมอื่น ๆ ได้ทันที:

1. **ต่อยอดฐานข้อมูลขึ้น Cloud (API Integration):**
   * หากนักพัฒนาต้องการให้แอพซิงก์ข้อมูลข้ามมือถือและคอมพิวเตอร์ผ่านการล็อกอิน สามารถเปลี่ยนส่วนฟังก์ชัน `loadData()` และ `saveData()` จากการเรียก `localStorage` ให้เป็นการเรียกใช้ Fetch API ส่งไปยังคลาวด์ภายนอก เช่น **Supabase, PocketBase หรือ Firebase**
2. **ปรับแต่งเช็คลิสเพิ่มเติม:**
   * สามารถเปิดไฟล์ `app.js` และพิมพ์ข้อความใส่ไอเทมย่อยในกลุ่มตัวแปร `DEFAULT_CATEGORIES` โค้ดจะนำเข้าไปคำนวณและแสดงผลให้ทันทีโดยไม่ต้องไปยุ่งเกี่ยวกับระบบ DOM
3. **การเพิ่ม PWA Manifest:**
   * หากสร้างไฟล์ `manifest.json` และติดตั้ง Service Worker เพิ่มเติม โปรเจกต์นี้จะกลายเป็น Progressive Web App (PWA) เต็มรูปแบบที่ทำงานแบบออฟไลน์ 100% ได้ทันที
